### How I think about AI

AI is now part of how I work, not just something I build.

I use AI tools daily to move faster across coding, debugging, documentation, project planning, and internal enablement. I've built agents that monitor Azure logs, summarize project timelines, surface errors with suggested fixes, and turn recurring operational work into reviewable workflows.

One habit I'm especially proud of: after each major project, I write a completion document covering the customer, problem, solution, estimate vs. actuals, and roadblocks. Those docs are ingested into our internal AI system, creating a searchable context layer for team velocity, recurring pain points, and future project planning.

I also run recurring AI training for executives, engineers, and nontechnical employees on practical AI adoption, from better prompting to agentic workflows.
