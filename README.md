### How I think about AI

AI is woven into pretty much everything I do at this point.

I use Claude Code every day for anything code-related. I've built agents that watch our Azure logs and surface errors with suggested fixes, Copilot agents that handle simple UI updates and bug fixes from ServiceNow tickets and open PRs for me to review, and a weekly email summary agent that tracks timelines across all my active projects.

The thing I'm most proud of is more of a long game. Every time a project ships, I write a project completion document covering the customer, problem, solution, time estimate vs. actuals, and roadblocks. Those go into our internal file system with AI ingestion turned on, so over time we're building a searchable context map of our team's velocity, where we tend to underestimate, and what customer pain points keep coming up. It compounds.

I block at least an hour a week to work on automating my own job. Sometimes that's a new tool's free trial, sometimes it's setting up an agent for whatever routine task is boring me that week. The compounding effect of that hour is probably the single highest-ROI thing on my calendar.
